## Short Blurb

Built a small local dashboard for understanding how Codex usage is concentrating across projects, task types, `xhigh`, planning, and subagents.

Codex already shows overall usage, but I wanted a clearer view of where the burn was actually coming from: repeated expensive patterns, heavy sessions, daily burn, and project mix. This reads local Codex history from `~/.codex`, stays local, and serves a browser dashboard from your machine.

## Alternative Blurb

I use Codex heavily across multiple projects and wanted a better local view than the native usage board alone. This tool groups sessions by inferred project and task type, surfaces repeated expensive setups, shows where `xhigh` / planning / subagents are concentrated, and makes daily burn easier to inspect.
